## 2. Design Philosophy — Event-Centric

Same 30 CQs as v2 (object-centric) but modeled from the event perspective. Every concept is expressed as an event that happens to or involves the object.

| Aspect | v2 (object-centric) | v3 (event-centric) |
|---|---|---|
| **Primary anchor** | `ArchaeologicalObject` class | Event classes (`ProductionEvent`, `UseEvent`, etc.) |
| **Material** | `hasMaterial` on object | `usesMaterial` on `ProductionEvent` |
| **Dimensions** | `hasLength/Width/Height` on object | `MeasurementEvent` with data properties |
| **Lifecycle** | Object has event links | `LifecycleChain` of ordered events |
| **Classification** | `TypologicalAssignment` (E17 pivot) | `ClassificationEvent` (event) |
| **Reuse** | `wasReusedIn` from object | `ReuseEvent` subclass of `crm:E7_Activity` |
| **Provenance** | Object → Source → Region | `ExtractionEvent` → `TransportEvent` → `ProductionEvent` chain |
| **Object agency** | `ObjectAgency` class | `AgencyExpressionEvent` affecting actors |

### Object participation pattern

Objects participate in events through properties:
- `participatedInProduction` → `ProductionEvent`
- `participatedInReuse` → `ReuseEvent`
- `participatedInRepair` → `RepairEvent`
- `participatedInCirculation` → `CirculationEvent`
- `participatedInConservation` → `ConservationEvent`
- `participatedInSampling` → `SamplingEvent`
- `participatedInRepatriation` → `RepatriationEvent`
