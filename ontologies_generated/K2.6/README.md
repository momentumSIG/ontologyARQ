## 2. Design Philosophy — Object-Centric vs Event-Centric

K2.6 adopts an **object-centric** design that differs from the deepseek-v4-pro event-centric approach:

| Aspect | Deepseekv4 (Event-centric) | K2.6 (Object-centric) |
|---|---|---|
| **Core entities** | Events (ProductionEvent, UseEvent, DepositionEvent, RecoveryEvent) | Objects (NaturalObject, HumanMadeObject, Artefact, Structure, CompositeObject) |
| **Hierarchy depth** | Flat (objects + events as siblings) | Deep (taxonomic tree: ArchaeologicalObject → NaturalObject/HumanMadeObject → AbioticObject/BioticObject/Artefact/Structure/ArtisticExpression) |
| **Reification** | 4 pivot classes (RecoveryContext, MaterialAssignment, TypologicalAssignment, BiographyLink) | Minimal; prefers direct attributes |
| **Physical attributes** | Minimal | Extensive (morphometry, weight, dimensions, completeness, Munsell color, grain size) |
| **Documentation** | PostExcavationTreatment (event) | DigitalDocumentation, PhotogrammetricModel, ScholarlyReference (entities) |
| **Stratigraphy** | DepositionEvent, RecoveryEvent (events) | StratigraphicSequence, NegativeFeature, Paleosol, ConstructionElement (entities) |

### Key design decision

The object-centric model treats the **archaeological object** as the primary ontological anchor. Events (production, use, deposition) are not modelled as explicit classes but implied through the object's relationships, functions, and contexts. This mirrors traditional archaeological artefact analysis where the object itself — its material, form, function, and context — is the analytical focus.
