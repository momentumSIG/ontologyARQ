## 2. Design Philosophy — Object-Centric vs Previous Event-Centric

Deepseekv4's second generation adopts an **object-centric** design, inverting its original event-centric approach.

| Aspect | Original Deepseekv4 (Event-centric) | Deepseekv4 v2 (Object-centric) |
|---|---|---|
| **Philosophy** | Events are the primary entities; objects participate in events | Objects are the primary entities; events happen to objects |
| **Core anchor** | Event classes (ProductionEvent, UseEvent, DepositionEvent) | Object class (ArchaeologicalObject) with hierarchy |
| **Hierarchy** | Flat (objects + events as siblings) | 2-3 level taxonomic tree with disjointness |
| **Classification** | Implicit | Explicit via E17_Type_Assignment reification |
| **Lifecycle modeling** | Event chain from production to recovery | Object lifecycle expressed through object-event links |
| **Physical properties** | Minimal | Material, dimensions, weight, form, completeness |
| **Material provenance** | Not modeled | RawMaterialSource, ProvenanceRegion, CirculationEvent |
| **Object agency** | Not modeled | ObjectAgency class, exertsAgency property |
| **Pastness/Patina** | Not modeled | Pastness and Patina as cultural value classes |
| **Repatriation** | Not modeled | RepatriationEvent with provenance and legal dimensions |

### Object classification taxonomy (temperature 0.7)

```
crm:E19_Physical_Object
└── arqo:ArchaeologicalObject
    ├── arqo:NaturalObject
    │   ├── arqo:AbioticObject
    │   └── arqo:BioticObject
    └── arqo:HumanMadeObject
        ├── arqo:Artefact
        ├── arqo:Structure
        └── arqo:ArtisticExpression
```

### Disjointness axioms (temp 0.7)

- `arqo:NaturalObject owl:disjointWith arqo:HumanMadeObject`
- `arqo:AbioticObject owl:disjointWith arqo:BioticObject`
- `arqo:Artefact owl:disjointWith arqo:Structure`

## 5. Temperature Comparison

### Temperature effect on design

| Aspect | 0.3 (Conservative) | 0.5 (Balanced) | 0.7 (Creative) |
|---|---|---|---|
| **Class count** | 1-2 core classes per CQ | 3-5 classes per CQ | 6-8 classes per CQ |
| **Object hierarchy** | Single class only | 2 levels (Natural/HumanMade) | 3 levels with 8 classes |
| **CRM reuse** | Maximum (E19, E57) | Standard | Extensive (E17, S19, S18, I4) |
| **Disjointness** | None | None | 3 axioms |
| **Event classes** | None | ReuseEvent, RepairEvent, CirculationEvent | Full lifecycle + RepatriationEvent |
| **Cultural concepts** | None | None | Pastness, Patina, ObjectAgency, CulturalSignificance |
| **Provenance** | None | RawMaterialSource | + ProvenanceRegion, CirculationEvent |

### Contrast with original Deepseekv4 event-centric design

| Aspect | v1 (event-centric) | v2 (object-centric) |
|---|---|---|
| First class defined | `ArchaeologicalObject` as subclass of `crm:E22_Human-Made_Object` | `ArchaeologicalObject` as subclass of `crm:E19_Physical_Object` |
| Early properties | Participatory events (participatedInProduction) | Material and physical properties (hasMaterial, hasLength) |
| Object taxonomy | None at 0.3 | NaturalObject/HumanMadeObject at 0.5 |
| Disjointness | None at any temperature | 3 axioms at 0.7 |
| Typological classification | Not modeled | TypologicalAssignment (E17_Type_Assignment) |
| Core philosophy | "What happened to it?" | "What is it?" |
