## 3. CRMarchaeo Integration Map

The `arqo:` module extends CRMarchaeo by connecting archaeological objects to stratigraphic units, depositional events, excavation processes, and taphonomic modifications.

### Namespace

```
arqo: <http://www.ontologyARQ.org/archaeological-object/>
```

### Class alignment with CRMarchaeo

The Archaeological Object module does **not** redefine CRMarchaeo concepts. It creates new subclasses that specialise CRMarchaeo classes for archaeological object-specific semantics.

```turtle
# в”Ђв”Ђ Stratigraphic Genesis в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
arqo:DepositionEvent    rdfs:subClassOf crmarchaeo:A4_Stratigraphic_Genesis .
# Links an archaeological object to the event that deposited it into a stratigraphic context.
# Extends A4_Stratigraphic_Genesis to capture the moment an object enters the archaeological record.

# в”Ђв”Ђ Excavation Process в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
arqo:RecoveryEvent      rdfs:subClassOf crmarchaeo:A1_Excavation_Process_Unit .
# Links an object to the excavation event that recovered it.
# Extends A1_Excavation_Process_Unit to track object-specific recovery.

# в”Ђв”Ђ Stratigraphic Modification в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
arqo:TaphonomicProcess  rdfs:subClassOf crmarchaeo:A5_Stratigraphic_Modification_Event .
# Models post-depositional processes (natural or anthropogenic) that alter the object.
# Extends A5_Stratigraphic_Modification_Event for taphonomy.

# в”Ђв”Ђ Physical Object в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
arqo:ArchaeologicalObject rdfs:subClassOf crm:E19_Physical_Object .
# The central class вЂ” a physical object recovered during archaeological investigation.
# Subclass of CRM's E19_Physical_Object, providing a domain-specific anchor.
```

### Key integration principle

CRMarchaeo models **excavation processes and stratigraphy**. The `arqo:` module models **archaeological objects**: what they are made of, how they were produced, used, reused, deposited, recovered, analysed, and interpreted. The two connect through:

1. **Class subsumption**: `arqo:` classes are subclasses of CRMarchaeo/CIDOC CRM classes
2. **Property ranges**: `arqo:` properties point to CRMarchaeo classes (e.g. `A8_Stratigraphic_Unit`)
3. **Event chains**: The object lifecycle (`Production в†’ Use в†’ Reuse в†’ Deposition в†’ Recovery в†’ Analysis`) maps to the CRM event-centric pattern

## 2. Design Philosophy вЂ” Object-Centric vs Event-Centric

K2.6 adopts an **object-centric** design that differs from the deepseek-v4-pro event-centric approach:

| Aspect | Deepseekv4 (Event-centric) | K2.6 (Object-centric) |
|---|---|---|
| **Core entities** | Events (ProductionEvent, UseEvent, DepositionEvent, RecoveryEvent) | Objects (NaturalObject, HumanMadeObject, Artefact, Structure, CompositeObject) |
| **Hierarchy depth** | Flat (objects + events as siblings) | Deep (taxonomic tree: ArchaeologicalObject в†’ NaturalObject/HumanMadeObject в†’ AbioticObject/BioticObject/Artefact/Structure/ArtisticExpression) |
| **Reification** | 4 pivot classes (RecoveryContext, MaterialAssignment, TypologicalAssignment, BiographyLink) | Minimal; prefers direct attributes |
| **Physical attributes** | Minimal | Extensive (morphometry, weight, dimensions, completeness, Munsell color, grain size) |
| **Documentation** | PostExcavationTreatment (event) | DigitalDocumentation, PhotogrammetricModel, ScholarlyReference (entities) |
| **Stratigraphy** | DepositionEvent, RecoveryEvent (events) | StratigraphicSequence, NegativeFeature, Paleosol, ConstructionElement (entities) |

### Key design decision

The object-centric model treats the **archaeological object** as the primary ontological anchor. Events (production, use, deposition) are not modelled as explicit classes but implied through the object's relationships, functions, and contexts. This mirrors traditional archaeological artefact analysis where the object itself вЂ” its material, form, function, and context вЂ” is the analytical focus.
