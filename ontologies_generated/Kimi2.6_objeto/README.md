### Temperature as experimental variable

| Temperature | Behaviour | Effect on output |
|---|---|---|
| **0.3** | Conservative | Minimal new classes, maximum CIDOC CRM reuse, flat hierarchies, no restrictions unless essential |
| **0.5** | Balanced | Moderate new classes, some restrictions, standard CRM alignment, reification where justified, hierarchical classes appear |
| **0.7** | Creative | Deep class hierarchies (3+ levels), reification pivot classes, extensive restrictions, disjointness axioms, full lifecycle event chain |

## 3. Design Philosophy — Event-Centric

Kimi-2.6 adopts an **event-centric** design that treats archaeological events as the primary ontological anchor:

| Aspect | Deepseekv4_objeto (v2) | Deepseekv4_objeto_eventos (v3) | Kimi-2.6 (this module) |
|---|---|---|---|
| **Philosophy** | Object-centric | Event-centric (same 30 CQs) | **Event-centric (new 30 CQs)** |
| **Core entities** | Objects (33 classes) | Events (17 classes) | **Events (121 classes @ 0.7)** |
| **Object role** | Primary anchor | Participant in events | **Participant in events** |
| **Event hierarchy** | Shallow (1-2 levels) | Intermediate (2 levels) | **Deep (3-4 levels @ 0.7)** |
| **Disjointness axioms** | None | None | **Extensive @ 0.7** |
| **Reification** | Minimal | Moderate | **Extensive @ 0.7** |
